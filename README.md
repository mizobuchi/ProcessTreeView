# EDR CSV Process Tree Viewer

CSVからプロセスツリーを作成
オフライン＆ブラウザだけで動きます

## できること

- ローカルの CSV をブラウザ上だけで解析
- サーバ不要、インターネット不要
- 数百MB級 CSV を想定したチャンク読み込み + Web Worker 解析
- 先頭10行プレビュー
- ヘッダ有無の指定
- `ProcessID` / `InitiatingProcessID` / `InitiatingProcessParentID` の3列を GUI で指定
- イベント時刻列 / プロセス開始時刻列 / イベント種別列 / ホスト列を任意指定
- 作成イベント値 / 終了イベント値を任意指定
- ログ順を「自動判定 / 古い順 / 新しい順」から選択
- 表示項目の選択
- ノード詳細では、表示項目で未選択の列を含む**代表行の全列**を表示
- プロセスツリーの拡大 / 縮小 / 移動 / ノード詳細表示
- 設定ペインの開閉（閉じるとツリー表示を広く使える）
- 右側のノード情報パネルの開閉（別ノード選択時は自動再表示）
- ノードごとに「子がもう無いのか / 上限で省略されたのか / 検索条件外なのか」を表示
- 最大ノード数の変更時に、現在の表示をその場で再描画
- 複数キーワード検索（OR / AND）と、該当ノードに関連する部分木だけの表示
- PID 再利用を考慮した親子接続
- 親プロセスがログ内に見つからない場合の**推定親ノード**表示

## ProcessID 系 3列の扱い

行ごとの有効 PID / 親 PID は次のルールで解決します。

- `ProcessID` に値がある行
  - 子プロセス = `ProcessID`
  - 親プロセス = `InitiatingProcessID`
  - その行を**作成行**として扱う
- `ProcessID` が空の行
  - 子プロセス = `InitiatingProcessID`
  - 親プロセス = `InitiatingProcessParentID`

## PID 再利用の解決ルール

優先順位は次のとおりです。

1. **プロセス開始時刻列** がある場合はそれを最優先
2. 開始時刻列がなければ **ProcessID が入った作成行** または **作成イベント** を開始点として利用
3. それもなければ **イベント時刻 + ログ順** で近似推定
4. 時刻情報が乏しい場合は **行順ベース** の低信頼推定

画面上では、各ノードの信頼区分を色で分けています。

- 緑: 開始時刻列で識別
- 青: 作成イベントで識別
- 黄: イベント時刻 / ログ順で推定
- 赤: 行順のみで推定
- 紫: 推定親（ログ外補完）

## 推定親ノード

ログが端末起動時点から始まっていない場合、子の `PPID` は分かっていても、その親実体の開始点が CSV 内に無いことがあります。

その場合、このツールは親未解決で切らずに、次の性質を持つ**推定親ノード**を生成して表示します。

- 既知の親実体が1つも無い場合は「未観測の親」として補完
- 同じ PID の既知親実体が後に現れる場合は「その前世代」として補完
- 同じ PID の既知親実体が複数ある場合は「世代の間の欠落区間」として補完

推定親ノードは元の CSV 行を持たないため、詳細パネルには「推定ノード」である旨と補完理由を表示します。

## 検索

- 検索欄には 1 行 1 キーワード、またはカンマ区切りで複数指定できます。
- `OR` は **いずれかのキーワードを含むノード** をヒットさせます。
- `AND` は **すべてのキーワードを含むノード** をヒットさせます。
- 検索対象は、各ノードの代表行だけではなく、**そのノードに属する全行から集約した文字列**です。
- 検索表示では、ヒットしたノードに加え、その**祖先と子孫**を表示します。

## 使い方

1. `index.html` をブラウザで開く
2. CSV を選択
3. 「プレビュー読込」を押す
4. `ProcessID` / `InitiatingProcessID` / `InitiatingProcessParentID` などの列を指定
5. 必要なら表示項目を選択
6. 「解析開始」を押す
7. 必要に応じて設定ペイン左上の「畳む」で閉じ、左上の「設定を開く」で再表示する
8. 右側の「選択ノード」パネルも畳めます。別のノードをクリックすると自動で開きます。
9. ルート候補または検索で表示対象を切り替える
10. 最大ノード数を変えると、現在の表示がその値で再描画される

## 推奨ブラウザ

- Chromium 系ブラウザ（Chrome / Edge）

大きな CSV を扱う場合、メモリ量の多い環境での利用を推奨します。

## 制限事項

- CSV 方言が極端に特殊な場合は、区切り文字や文字コードを手動指定してください。
- 時刻列が存在しない、または作成行 / 作成イベントが存在しないログでは、PID 再利用判定は近似になります。
- 推定親ノードは、欠落した親実体を補うための近似表現です。ログ外の真の親開始時刻を復元するものではありません。
- 非常に巨大なツリーは、描画時に最大ノード数で切り詰めます。
- ノード詳細の全列表示は、**そのノードの代表行**に基づきます。推定親ノードには対応する元行がありません。
