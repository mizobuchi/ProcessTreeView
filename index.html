EDR Process Tree Viewer

ブラウザだけで動作するオフライン版。CSVはローカルでのみ処理され、サーバ送信しません。

数百MB級のCSVを想定し、チャンク読み込み + Web Workerで解析します。大きなファイルでは、まず代表的な列だけを表示項目として選ぶと扱いやすくなります。

1. ファイルとプレビュー

ファイルを選択してプレビューを読み込んでください。

2. フィールド指定

Microsoft Defender系の列構成を想定しています。ProcessID / InitiatingProcessID / InitiatingProcessParentID の3列は必須です。親子関係の解決では、まず各行に含まれる作成時刻列を優先し、無ければイベント時刻＋ログ順で近似します。
イベント時刻フィルタは解析結果を保持したまま表示だけを絞り込みます。入力は「列の時刻タイムゾーン（認識用）」基準です。判定はノードが保持する時刻窓（開始 / 終了 / earliest / latest）を使うため、イベント時刻を行単位で全件保持する方式よりメモリを抑えられますが、長時間動作したプロセスでは範囲内に実イベントが無くても表示される場合があります。
時刻値に Z や +09:00 などのタイムゾーン表記がある場合はその値を優先し、無い場合だけ下の認識設定を使います。表示タイムゾーンの変更は再解析不要で即時反映されます。

3. 表示項目

ここで選んだ列をノード表示と詳細パネルに反映します。解析コストを抑えるため、必要な列だけを選ぶのがおすすめです。

4. 解析

まだ解析していません。
開始時刻列で識別 作成イベントで識別 イベント時刻 / ログ順で推定 行順だけで推定

5. ルート選択 / 検索 / 表示フィルタ

再起動などでルートが複数ある場合は「全ルート」を選ぶと複数のツリーをまとめて表示できます。表示フィルタは一致ノード + その祖先 + その子孫だけを残し、兄弟や無関係の枝を隠します。
