# MDE Process Tree Builder

MDE の CSV / TSV ログから、ブラウザだけでプロセスツリーを作る静的ツールです。

## 使い方

1. `index.html` をブラウザで開く。
2. MDE のログファイルを選択またはドラッグする。
3. 自動判定された列マッピングを確認する。
4. `解析` を押す。

`サンプル` ボタンで、ログなしでも動作確認できます。

## 対応の考え方

- 列順には依存せず、ヘッダー名から列を判定します。
- UTF-8 / UTF-16 BOM 付きログを自動判定します。
- `ProcessUniqueId` / `InitiatingProcessUniqueId` があれば優先します。
- 一意 ID がない場合は `Device`、`ProcessId`、`ProcessCreationTime` を優先してプロセスを識別します。
- 親プロセスがログ外の場合は、`InitiatingProcess*` または `ParentProcess*` 列から推定親ノードを作成します。
- 同じ推定親は同一ノードにまとめます。
- 親情報がないルートは、設定によりデバイス単位の `[親不明]` ノードにまとめます。
- 大きなファイルに備えて、読み込みと集計は Web Worker でストリーミング処理します。
- ツリーにはノードごとの `ActionType` サマリを表示します。
- Details には `ActionType` ごとの元ログ行番号を表示し、行番号を選ぶとそのイベントの全列データを表示します。イベントが1件だけのノードでは、その行を自動選択します。

## 注意

- 画面表示用のノード数は上限で絞ります。解析自体は全行を読みます。
- ブラウザのメモリ上限を超える規模では、表示ノード上限を下げてください。
- 入力ログはローカルブラウザ内で処理され、サーバーには送信しません。
