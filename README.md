# EDR CSV Process Tree Viewer

ブラウザだけで動作する、オフライン前提のデジタルフォレンジック補助ツールです。CSV はローカルでのみ処理され、サーバやインターネット接続は不要です。

## 主な機能

- 数百MB級 CSV を想定したチャンク読み込み + Web Worker 解析
- 先頭 10 行プレビュー、ヘッダ有無指定、区切り文字と文字コード指定
- `ProcessID` / `InitiatingProcessID` / `InitiatingProcessParentID` の 3 列指定
- `Process Creation Time` / `Initiating Process Creation Time` / `Initiating Process Parent Creation Time` の任意指定
- イベント時刻列、イベント種別列、ホスト / デバイス識別列の任意指定
- 列の時刻タイムゾーン認識設定と表示タイムゾーン設定（表示デフォルトは JST）
- PID 再利用を考慮した親子推定
- ログに create が無い親も、PPID から **推定親ノード** として補完表示
- 同一 scope / 同一 PPID / 同一世代スロットの未観測親は **1 つの推定親ノードに統合**
- 複数ルート検出時の「全ルート」表示
- 左の設定ペイン、各設定セクション、右の詳細ペイン、右ペイン内カードの開閉
- ツリーの拡大 / 縮小 / パン / ノード選択
- ノード詳細で代表行の全項目を表示
- ノードの基本情報に元ファイル行番号（範囲 / 代表 / 開始判定 / 終了）を表示
- ノード選択時に、そのプロセスに紐づく **ProcessCreate 以外の関連イベント** を右ペイン一覧に表示
- 関連イベント一覧クリックで、**選択イベント詳細** に全項目を表示
- 選択ノード直下に、関連イベントを **四角ノード** として縦並び表示（イベント種別 + 表示項目ベースの内容）
- ノードのない部分をクリックすると選択解除し、関連イベント表示も消去
- 検索機能（PID / ラベル / 表示項目ベース）
- 表示フィルタ（複数キーワード、AND / OR、表示タイムゾーン基準の時刻範囲）

## PID 系列の扱い

- `ProcessID` に値がある行: 子 = `ProcessID`、親 = `InitiatingProcessID`
- `ProcessID` が空の行: 子 = `InitiatingProcessID`、親 = `InitiatingProcessParentID`
- `ProcessID` に値がある行は、開始時刻列が無くてもプロセス作成行として扱います

## 親子関係の解決

親子解決の優先順位は次のとおりです。

1. 作成時刻列が指定されていればそれを優先
2. 作成イベント値が指定されていれば作成イベントを開始点として使用
3. それも無ければイベント時刻 + ログ順で近似
4. 時刻情報が乏しければ行順ベースで低信頼推定
5. それでも親が観測されていなければ、PPID から推定親を補完

## 解析と描画上限

- **解析は描画上限と独立して全件実行**します
- `描画時の最大ノード数` は **表示時だけ** に適用されます
- 固定上限は設けていません
- 値が大きい場合は確認ダイアログを出し、ブラウザが重くなる可能性を明示します
- **表示時刻フィルタ** は解析済みノード集合に対して **表示時だけ** に適用されます
- **表示タイムゾーンの変更**は再解析不要で、表示と詳細だけを更新します

## 表示時刻フィルタの注意点

表示時刻フィルタは、全件解析後に保持している各ノード / 関連イベントのイベント時刻を使って判定します。入力は **表示タイムゾーンの時刻** です。

- フィルタは再解析ではなく **再描画** で反映されます
- ノードごとのイベント時刻一覧を保持するため、**大きな CSV ではブラウザのメモリ使用量が増えます**
- 直接一致ノードの **祖先 / 子孫（孫以降を含む）** も表示します
- 関連イベント一覧とツリー直下の関連イベントノードにも同じ時刻フィルタが反映されます
- 表示上は範囲外の祖先がコンテキストとして残ることがあります

## 表示フィルタ

- 複数キーワードを改行 / カンマ / セミコロン区切りで指定できます
- 表示時刻フィルタは **表示タイムゾーン** の日時で入力します
- `OR` はいずれか一致、`AND` はすべて一致です
- 直接一致したノードに加えて、その **祖先** と **子孫（子・孫・さらにその先まで再帰的）** を残します
- 一致していない **兄弟ノード** や無関係の枝は非表示になります

## 使い方

設定ペインと右ペインは、境界線をドラッグして幅を変更できます。


1. `index.html` をブラウザで開く
2. CSV を選択して「プレビュー読込」を実行
3. 必須列と任意列を指定
4. 必要なら表示フィルタ（キーワード / 時刻）とタイムゾーン設定を調整
5. 「解析開始」を押す
6. ルート候補から対象を選んで表示する
7. ノードをクリックして右ペインで詳細を確認する
8. 必要に応じて、関連イベント一覧またはノード直下の四角ノードをクリックしてイベント詳細を確認する

## 配布物の検証

同梱の `verify_release.sh` では、次を確認します。

- 単体HTMLとZIP内HTMLの一致
- 開閉UIと関連イベントUIの存在
- Worker / メインスクリプトの構文チェック
- PPID からの推定親ノード補完
- 関連イベント取得 (`node-detail` / `node-events`) の簡易実行テスト

## 推奨ブラウザ

- Chromium 系ブラウザ（Chrome / Edge）

大きな CSV を扱う場合、メモリ量の多い環境での利用を推奨します。

## リリース検証

同梱の `verify_release.sh` では、次を確認します。

- 単体 `index.html` と ZIP 内 `index.html` の一致
- 設定ペイン / 右ペイン / 畳み UI の存在
- PPID からの推定親補完、関連イベント UI、描画上限警告の存在
- 埋め込み Worker / メインスクリプトの構文チェック
- 小さな回帰テストデータによる実行確認
  - PPID からの推定親が補完されること
  - 同じ未観測親が 1 ノードに統合されること
  - ProcessCreate 以外の関連イベントが保持されること
  - 表示フィルタで一致ノードの子孫が残ること


- 左の設定ペインと右の詳細ペインは、境界線をドラッグして幅を変更できます
