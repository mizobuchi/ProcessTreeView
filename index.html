EDR Process Tree Viewer

ブラウザだけで動作するオフライン版。CSVはローカルでのみ処理され、サーバ送信しません。

数百MB級のCSVを想定し、チャンク読み込み + Web Workerで解析します。大きなファイルでは、まず代表的な列だけを表示項目として選ぶと扱いやすくなります。

1. ファイルとプレビュー

ファイルを選択してプレビューを読み込んでください。

2. フィールド指定

3つのPID系列を指定します。ProcessID に値がある行はプロセス作成として扱い、親は InitiatingProcessID を使います。ProcessID が空の行は、子を InitiatingProcessID、親を InitiatingProcessParentID として扱います。開始時刻列があればそれを最優先し、なければ作成行、さらに無ければイベント時刻＋ログ順でPID再利用を近似解決します。

3. 表示項目

ここで選んだ列をノード表示に反映します。詳細パネルでは代表行の全項目を表示します。解析コストを抑えるため、必要な列だけを選ぶのがおすすめです。
ノード上の要約と右ペイン内の非作成イベント情報を表示/非表示できます。解析後も再解析なしで切り替えられます。

4. 解析

まだ解析していません。
開始時刻列で識別 作成イベントで識別 イベント時刻 / ログ順で推定 行順だけで推定

5. ルート選択 / 検索

プロセスツリー

ノードをクリックすると詳細を表示します。マウスホイールで拡大縮小、ドラッグで移動できます。親がログに無い場合は推定親を生成し、大きなツリーは最大ノード数で自動的に切り詰めます。再起動などでルートが複数ある場合は、初期表示で全ルートをまとめて表示できます。右ペインはビューア上に固定表示され、内容が長い場合はペイン内をスクロールして確認できます。

関連イベント一覧から項目を選択すると、ここにイベント詳細を表示します。
このツールは完全オフラインで動作します。ファイル内容はブラウザ内メモリにのみ保持され、外部送信しません。右ペインはツリー表示の上に重ねて表示され、画面サイズを超える内容だけ右ペイン内でスクロールして確認できます。選択ノード概要に個別の畳むボタンはありません。下の各セクションだけ個別に畳めます。必要に応じて、ProcessCreate以外のイベント情報を表示/非表示できます。解析直後は「全ルート」が選ばれるため、再起動などで root が複数ある場合も最初から複数のツリーを同時表示できます。
